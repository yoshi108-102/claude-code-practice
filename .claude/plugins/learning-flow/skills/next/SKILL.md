# 次タスクへの進行管理

現Taskの完了状況を確認したうえで、LEARNING_CONTEXT.md の進捗を更新し、次Taskの解説を始めるコマンド。

## 実行フロー

### ステップ1: 現Taskの特定

1. `docs/LEARNING_CONTEXT.md` から進行中Phaseを取得
2. `docs/learning/phase{N}/task*/` から進行中Task（`main.md` の振り返りが未記入）を特定

現Taskが見つからない場合:

```
進行中のTaskが見つかりません。以下のどちらですか？
(a) Task初回で、まだ main.md が作られていない → Task番号を指定してください
(b) 全Task完了 → /learning-flow:next は次Phaseへ進みます
```

### ステップ2: 完了チェック（3項目）

以下の3つを順にチェックし、不足があれば警告を出して中断する。

#### チェックA: トピックmdが1つ以上ある

`docs/learning/phase{N}/task{M}/[0-9][0-9]-*.md` を `Glob` で検索。

0件の場合:

```
警告: このTaskの学習トピックmdが保存されていません。
  解説や質疑応答を /learning-flow:topic で保存してから /learning-flow:next を実行してください。
  本当にこのまま次へ進みますか？（y/n）
```

ユーザーが `y` を返したら続行、`n` なら中断。

#### チェックB: main.md の振り返りがクイズ形式で記入されている

`docs/learning/phase{N}/task{M}/main.md` を `Read` し、「## 振り返り」セクション以下の状態を判定:

**状態1: 未生成（空 or「（Task完了時に記入）」のまま）**

→ **クイズを生成して main.md に書き込む**。対話だけで済ませず、必ずファイルに書き込むこと。

1. 会話履歴・保存済みトピックmd・実装ステップ中の解説内容から主要概念を抽出
2. クイズ問題を生成（合計8〜10問）:
   - **対話中のQ&Aから** 3〜5問（ユーザーが発した質問・混同しかけた概念・補足を要した箇所）
   - **実装・解説ステップで取り上げた概念から** 3〜5問（設計判断、コード上の選択、ベストプラクティス）
   - 問題の難易度は混ぜる（事実確認・理由説明・トレードオフ評価など）
3. main.md の「## 振り返り」セクションを以下の形式に書き換え:

```markdown
## 振り返り（クイズ形式）

回答は各問の下に記入。採点・補足は Claude が `/learning-flow:next` 実行時に行う。

### Q1. {{トピック名}}

{{問題文}}

### Q2. {{トピック名}}

{{問題文}}

...
```

4. ユーザーに案内:

```
main.md の「## 振り返り」にクイズを書き込みました（全{{N}}問）。
各問の下に回答を記入して、再度 /learning-flow:next を実行してください。
次回、回答を採点してから次Taskへ進みます。
```

→ この時点で `/learning-flow:next` は**中断**する。次Taskへは進めない。

**状態2: クイズが生成されているが回答が未記入**

各問の下に回答が書かれていない、または「分からない」「未回答」等のみの場合:

```
クイズの回答がまだのようです。各問の下に回答を記入してから再度 /learning-flow:next してください。
部分的でも「分からない」と書けば採点対象にします。
```

→ 中断。

**状態3: クイズに回答が記入されている**

→ **採点ステップへ進む**:

1. 各問の回答を読み、正誤・部分正解・誤答を判定
2. 各問について:
   - 正解 → 「✅ 正解」と1行コメント（深掘りが有益なら補足）
   - 部分正解 → 「⚠️ 一部正解」+ 不足している観点を補足
   - 誤答 → 「❌ 正確には...」+ 正しい考え方を提示
   - 「分からない/要復習」→ 採点せず解説して学習のフォローに回す
3. 採点結果を**チャットに出力**（main.md への書き込みは任意）
4. **復習キューへの追記**: ❌誤答・⚠️一部正解・「要復習」と書かれた問題を `docs/learning/review-queue.md` の「待機中」セクションに追記する

   追記フォーマット（各エントリ）:

   ```markdown
   ### [YYYY-MM-DD] Phase N / Task M — Q{番号} {短いタイトル}

   **問題**: ...
   **当時の回答**: ...
   **模範解答の要点**:
   - ...
   **関連ノート**: [path](path)
   ```

   - 日付は採点日（今日）
   - ファイルがなければ新規作成（テンプレートは [docs/learning/review-queue.md の先頭コメント](../../../../../competitive_programming_review/docs/learning/review-queue.md) を参照）
   - ✅ 正解の問題はキューに追加しない

5. ユーザーに確認:

```
採点完了。
- 復習キューに {N} 件追加しました（次回学習セッション開始時に再出題されます）
- この内容で次Taskへ進みますか？
(a) はい、進む
(b) いいえ、もう少し復習したい → どの問題を深掘りしますか？
```

### クイズ生成の指針

- **Q&A偏重にしない**: ユーザーが明示的に質問した論点だけで作ると、Claude側から解説した重要概念（ユーザーが自然に納得した部分）が抜ける
- **実装判断を問う問題を必ず入れる**: 「なぜこう書いたか」「別の選択肢の場合どうなるか」
- **混同しやすい用語の区別を問う**: 今回の例 = `user_id` vs AtCoder handle、name vs ARN
- **記憶より理解を問う**: 「〜とは何か」より「なぜ〜か」「もし〜だったら」を優先

#### チェックC: ユーザー確認

A, B を通過しても、最後に必ず確認:

```
Task {M} の完了を LEARNING_CONTEXT.md に反映し、Task {M+1} へ進みます。よろしいですか？
(a) はい、進む
(b) いいえ、中断
```

### ステップ3: 進捗の更新

ユーザー承認後、以下を `Edit` で更新:

#### A. LEARNING_CONTEXT.md の進捗テーブル

- 全Phase完了しうる場合、Phase 状態を更新:
  - 現Task が Phase 最後のTask → 該当Phase行の「進行中」を「完了」に
  - 次のPhase行の「未着手」を「進行中」に
- Phase内の進行なら、「現在の進捗」セクションを更新:

```markdown
## 現在の進捗

**Phase {N} / Task {M+1}**: {{次Taskのタイトル}} の学習段階。
```

#### B. 次Taskディレクトリの準備

`docs/learning/phase{N}/task{M+1}/` が存在しない場合、`Bash` で `mkdir -p` する。

`main.md` テンプレを新規作成:

```markdown
# Phase {N} / Task {M+1}: {{次Taskのタイトル}}

## 目次

（解説が進んだら /learning-flow:topic で追加）

## 振り返り

（Task完了時に記入）
```

### ステップ4: 次Taskの解説開始

`plans/*-design.md` から次Taskに該当するセクションを抽出し、ユーザーに提示:

```
次のTaskに進みます。

## Phase {N} / Task {M+1}: {{次Taskのタイトル}}

{{設計書から抜粋した Task の目的・ゴール・学ぶこと}}

---

まずは {{最初のサブトピック}} から解説しますか？
それとも別のトピックから始めたいですか？
```

設計書に次Taskの記述がない場合:

```
設計書 (plans/*-design.md) に Task {M+1} の記述が見つかりません。
以下のどちらで進めますか？
(a) 新しいTaskのテーマを口頭で聞いて進める
(b) 一度中断して設計書を追記してから再開する
```

## 重要な注意点

### 勝手に実装を始めない

次Taskの解説を「開始」するのみで、いきなりコードを書かない。`learning-flow-rules` の「解説→確認→実装」サイクルを守る。

### Phase完了時の特別扱い

現Taskが Phase最後のTaskで、かつ完了チェックを通過した場合:

1. Phase行の状態を「完了」に更新
2. ユーザーに Phase 全体の振り返りを促す:
   ```
   Phase {N} 完了。
   
   次Phaseに進む前に、Phase全体の振り返りを LEARNING_CONTEXT.md に残しませんか？
   - このPhaseで習得した概念
   - 当初の想定との差異
   - 次Phaseに活かせる学び
   
   書きますか？（y/n）
   ```
3. 承認後、Phase {N+1} の最初のTaskへ進む

### 全Phase完了時

最後のPhaseの最後のTaskを完了した場合:

```
全 {PHASE_COUNT} Phase 完了。おめでとうございます。

総仕上げとして、以下を提案します:
- LEARNING_CONTEXT.md に全体振り返りセクションを追加
- 最初の想定と実際の学びのギャップを書き出す
- 次に学びたいテーマの洗い出し
```

### 冪等性

ユーザーが誤って `/learning-flow:next` を2回連続実行した場合:

- 2回目の実行で「進行中」Taskが既に次Taskになっている → 「進捗は既に更新済みです。もう一度進めますか？」と確認
